# Project Local Standards Generation Canon

Workshop canons do not execute directly inside a project. They pressure the project to generate
local standards that fit its type, stack, domain, scale, and constraints.

## Rule

Universal standard -> project comparison -> local standard candidate -> accepted local truth -> execution.

## Local Standards May Include

- project truth map,
- stack or environment standard,
- architecture standard,
- domain workflow standard,
- data/API/evidence standard,
- testing and quality standard,
- security/governance standard,
- writing or content style standard,
- delivery and review gate.
