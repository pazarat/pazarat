# Workshop Operating Principles

The workshop is the silent universal knowledge gate of the project engineer environment.
It is not a replacement for active project truth. It is the method lens used to evaluate,
mature, and specialize any project before execution.

## Role

The workshop provides:
- universal project engineering method,
- 360-degree evaluation pressure,
- discipline-level canons,
- maturity and incomplete-truth detection,
- local standards generation rules,
- execution gates.

## Boundary

Workshop standards may reveal gaps and propose local standards, but they cannot directly
change project truth or override active project files. Execution must follow project-local
accepted truth.

## Operating Sequence

1. Enter the central plug.
2. Load the silent workshop lens.
3. Detect project type and maturity from active project files.
4. Use workshop canons to compare current reality against required maturity.
5. Generate or repair local project standards when required.
6. Execute only through local accepted truth.
