# Project Maturity Assessment Canon

Maturity is not file count, length, or attractive wording. Maturity is evidence that the project can
move safely from intent to delivery without guesswork.

## Assessment Dimensions

- root truth exists,
- project type is identified,
- owner files exist for key concepts,
- artifacts have enough semantic density,
- workflows and scenarios are clear,
- incomplete truth is detected rather than ignored,
- local standards exist for the project type,
- execution gates are clear,
- tests or acceptance criteria exist for delivery work,
- gaps have owners and repair paths.

## Output Requirement

Every maturity judgment must distinguish:
- observed fact,
- evidence-backed inference,
- assumption,
- open decision.
