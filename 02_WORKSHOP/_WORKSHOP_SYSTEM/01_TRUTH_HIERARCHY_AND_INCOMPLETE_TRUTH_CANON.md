# Truth Hierarchy and Incomplete Truth Canon

Truth is hierarchical. A project file may be binding, yet incomplete for a requested operation.

## General Hierarchy

1. Current user instruction.
2. Active project files.
3. Project contracts, indexes, entrypoints, and local tunnels.
4. Project artifacts and decisions.
5. Workshop universal method and discipline canons as maturity lenses.
6. Generated reports and indexes only when they do not conflict with current files.
7. Inference.
8. Declared assumptions.

## Incomplete Truth

A file, standard, PRD, or decision is incomplete truth when it exists and is binding, but lacks
required dimensions for the operation requested.

The correct response is not to ignore it and not to invent around it. The correct response is:
- identify the owner,
- classify the missing dimensions,
- compare against higher project truth and workshop canon,
- propose repair in the owner file or a local standard,
- block production-oriented execution until accepted.

## Rule

Discovery may expose incomplete truth and propose repairs. Execution must obey accepted local truth.
