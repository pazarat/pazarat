# وثيقة متطلبات المنتج (PRD): قسم التسويق بالعمولة (Affiliate Section)

## 1. المقدمة

يُعد قسم التسويق بالعمولة (Affiliate Section) ضمن موديول الأعمال (Business Module) في لوحة تحكم المسؤول هو المسؤول عن إدارة نظام التسويق بالعمولة المتكامل لمنصة "pazarat". يغطي هذا القسم تسجيل الوكلاء، إدارة خطط العمولات، تتبع أدائهم، وحساب العمولات المستحقة. تهدف هذه الوثيقة إلى توضيح الأهداف العامة لهذا القسم، أقسامه الفرعية، وكيفية ترابطها لضمان إدارة فعالة وشفافة لبرنامج التسويق بالعمولة.

## 2. الرؤية والأهداف

**الرؤية:** توفير نظام تسويق بالعمولة قوي ومرن يجذب الوكلاء، يحفزهم على الترويج للمنتجات، ويضمن الشفافية في حساب وتوزيع العمولات، مما يساهم في توسيع قاعدة العملاء وزيادة المبيعات.

**الأهداف:**

*   إدارة تسجيل الوكلاء ومراجعة طلبات الانضمام.
*   تحديد وإدارة خطط العمولات المختلفة (نسبة مئوية، مبلغ ثابت، لكل منتج).
*   تتبع أداء الوكلاء (النقرات، التحويلات، المبيعات).
*   حساب العمولات المستحقة للوكلاء بشكل دقيق وآلي.
*   توفير تقارير شاملة حول أداء برنامج التسويق بالعمولة.
*   إدارة المواد التسويقية المتاحة للوكلاء (روابط، لافتات).

## 3. الأقسام الفرعية والترابط

يتكون قسم التسويق بالعمولة من الأقسام الفرعية التالية، والتي تتفاعل فيما بينها لضمان سير العمليات:

### 3.1. جميع الوكلاء (All Affiliates)

*   **الهدف:** عرض قائمة شاملة بجميع الوكلاء المسجلين في برنامج التسويق بالعمولة، مع إمكانيات البحث، التصفية، وعرض التفاصيل الأساسية لكل وكيل.
*   **الترابط:** نقطة انطلاق لإدارة أي وكيل. يتكامل مع الأقسام الفرعية الأخرى (Pending Affiliates, Active Affiliates, Disabled Affiliates) لعرض حالة الوكيل المحددة.

### 3.2. الوكلاء المعلقون (Pending Affiliates)

*   **الهدف:** عرض قائمة بالوكلاء الذين قدموا طلبات للانضمام إلى البرنامج وينتظرون المراجعة والموافقة من قبل المسؤول. يسمح للمسؤول بمراجعة معلومات الوكيل واتخاذ قرار بالموافقة أو الرفض.
*   **الترابط:** يتكامل مع قسم التحقق (Business -> Users -> Verification) لضمان استيفاء الوكيل لمتطلبات التحقق قبل الموافقة.

### 3.3. الوكلاء النشطون (Active Affiliates)

*   **الهدف:** عرض قائمة بالوكلاء الذين تم تفعيل حساباتهم ويعملون حالياً في البرنامج. يسمح للمسؤول بمراقبة أدائهم، وتعديل معلوماتهم، أو تعطيلهم إذا لزم الأمر.
*   **الترابط:** يتكامل مع قسم تقارير الأداء (Performance Reports) لتتبع أداء الوكلاء النشطين.

### 3.4. الوكلاء المعطلون (Disabled Affiliates)

*   **الهدف:** عرض قائمة بالوكلاء الذين تم تعطيل حساباتهم لأسباب مختلفة (مثل عدم الامتثال للسياسات، الأداء الضعيف، أو طلب الوكيل). يسمح للمسؤول بمراجعة سبب التعطيل، وإعادة تفعيل الوكيل إذا تم حل المشكلة.
*   **الترابط:** يتكامل مع قسم الدعم (Marketing -> Support) للتعامل مع استفسارات الوكلاء بخصوص حساباتهم المعطلة.

### 3.5. خطط العمولات (Commission Plans)

*   **الهدف:** إدارة خطط العمولات المختلفة التي يمكن تطبيقها على الوكلاء. يشمل ذلك تحديد نسب العمولات (مئوية أو ثابتة)، المنتجات أو التصنيفات التي تنطبق عليها، وشروط الدفع.
*   **الترابط:** يتكامل مع قسم المنتجات (Business -> Products) لربط خطط العمولات بمنتجات محددة، ومع قسم العمولات (Financial -> Commission) لحساب العمولات بناءً على هذه الخطط.

### 3.6. تقارير الأداء (Performance Reports)

*   **الهدف:** توفير تقارير شاملة حول أداء الوكلاء، بما في ذلك عدد النقرات، التحويلات، المبيعات المحققة، والعمولات المكتسبة. يساعد في تقييم فعالية الوكلاء والبرنامج ككل.
*   **الترابط:** يتكامل مع موديول البيانات الذكية (Smart Data -> Reports) لتقديم تقارير مفصلة، ومع قسم العمولات (Financial -> Commission) لتتبع العمولات المستحقة.

### 3.7. المواد التسويقية (Marketing Materials)

*   **الهدف:** إدارة المواد التسويقية التي يمكن للوكلاء استخدامها للترويج للمنتجات، مثل الروابط المخصصة، اللافتات الإعلانية، أو النصوص الترويجية. يسمح للمسؤول بتحميل وتحديث هذه المواد.
*   **الترابط:** يتكامل مع قسم المنتجات (Business -> Products) لربط المواد التسويقية بمنتجات محددة.

## 4. الخلاصة

يُعد قسم التسويق بالعمولة عنصراً استراتيجياً في لوحة تحكم المسؤول، حيث يساهم في توسيع نطاق وصول المنصة وزيادة المبيعات. من خلال توفير أدوات شاملة لإدارة الوكلاء، خطط العمولات، وتقارير الأداء، يدعم هذا القسم النمو المستمر لمنصة pazarat ويضمن الشفافية في برنامج التسويق بالعمولة.
