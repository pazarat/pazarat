# وثيقة متطلبات المنتج (PRD): قسم إدارة المحتوى (CMS Section)

## 1. المقدمة

يُعد قسم إدارة المحتوى (CMS Section) ضمن موديول النظام (System Module) في لوحة تحكم المسؤول هو المسؤول عن إدارة جميع المحتوى الثابت والديناميكي لمنصة "pazarat". يغطي هذا القسم إنشاء الصفحات، المقالات، الأخبار، والتحكم في محتوى الأقسام المختلفة للمنصة. تهدف هذه الوثيقة إلى توضيح الأهداف العامة لهذا القسم، أقسامه الفرعية، وكيفية ترابطها لضمان إدارة مرنة وشاملة للمحتوى.

## 2. الرؤية والأهداف

**الرؤية:** توفير نظام إدارة محتوى قوي ومرن يمكن المسؤولين من تحديث وإدارة محتوى المنصة بسهولة، مما يضمن تحديث المعلومات، ويزيد من جاذبية المنصة، ويدعم استراتيجيات التسويق والمحتوى.

**الأهداف:**

*   إنشاء وإدارة الصفحات الثابتة (مثل: من نحن، سياسة الخصوصية، شروط الاستخدام).
*   إدارة المقالات والأخبار والمدونات.
*   التحكم في محتوى الأقسام الديناميكية (مثل: نصوص الترحيب، رسائل الخطأ).
*   توفير محرر نصوص غني (Rich Text Editor) لسهولة إنشاء المحتوى.
*   إدارة إصدارات المحتوى والقدرة على استعادة النسخ السابقة.
*   دعم المحتوى متعدد اللغات.

## 3. الأقسام الفرعية والترابط

يتكون قسم إدارة المحتوى من الأقسام الفرعية التالية، والتي تتفاعل فيما بينها لضمان سير العمليات:

### 3.1. جميع الصفحات (All Pages)

*   **الهدف:** عرض قائمة شاملة بجميع الصفحات الثابتة والديناميكية في المنصة، مع إمكانيات البحث، التصفية، وعرض التفاصيل الأساسية لكل صفحة (الاسم، العنوان، الحالة، تاريخ آخر تحديث).
*   **الترابط:** نقطة انطلاق لإدارة أي صفحة. يتكامل مع الأقسام الفرعية الأخرى (Create New Page, Page Details, Page History) لتنفيذ الإجراءات على الصفحات.

### 3.2. إنشاء صفحة جديدة (Create New Page)

*   **الهدف:** توفير واجهة لإنشاء صفحات جديدة. يشمل ذلك إدخال عنوان الصفحة، المحتوى (باستخدام محرر نصوص غني)، تحديد القالب، وتحديد حالة النشر.
*   **الترابط:** يتكامل مع قسم إعدادات اللغة (System -> Settings -> Language Settings) لدعم المحتوى متعدد اللغات.

### 3.3. إدارة المقالات (Articles Management)

*   **الهدف:** إدارة المقالات والأخبار والمدونات. يشمل ذلك إنشاء مقالات جديدة، تحرير المقالات الموجودة، وتحديد فئات المقالات.
*   **الترابط:** يتكامل مع قسم الحملات (Marketing -> Campaigns) لربط المقالات بالحملات التسويقية.

### 3.4. إدارة الأصول (Assets Management)

*   **الهدف:** إدارة الصور والفيديوهات والملفات الأخرى المستخدمة في المحتوى. يسمح بتحميل الأصول وتضمينها في الصفحات والمقالات.
*   **الترابط:** يتكامل مع قسم مدير الملفات (Smart Data -> File Manager) لإدارة الأصول الرقمية.

### 3.5. سجل التغييرات (Content History)

*   **الهدف:** عرض سجل كامل لجميع التغييرات التي طرأت على المحتوى، بما في ذلك من قام بالتغيير ومتى. يسمح باستعادة النسخ السابقة من المحتوى.
*   **الترابط:** يتكامل مع قسم سجلات الوصول (System -> Access Control Logs) لتوفير رؤية شاملة للأنشطة الإدارية.

## 4. الخلاصة

يُعد قسم إدارة المحتوى عنصراً حيوياً في لوحة تحكم المسؤول، حيث يضمن إدارة مرنة وشاملة لجميع المحتوى في منصة pazarat. من خلال توفير أدوات شاملة لإنشاء، تحرير، وتنظيم المحتوى، يدعم هذا القسم تحديث المعلومات ويساهم في جاذبية المنصة ونموها المستمر.
