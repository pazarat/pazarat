# وثيقة متطلبات المنتج (PRD): قسم السائقين (Drivers Section)

## 1. المقدمة

يُعد قسم السائقين (Drivers Section) ضمن موديول العمليات (Operations Module) في لوحة تحكم المسؤول هو المسؤول عن إدارة جميع السائقين الذين يعملون مع منصة "pazarat" لتوصيل الطلبات. يغطي هذا القسم تسجيل السائقين، إدارة ملفاتهم الشخصية، تتبع أدائهم، وتوزيع المهام عليهم. تهدف هذه الوثيقة إلى توضيح الأهداف العامة لهذا القسم، أقسامه الفرعية، وكيفية ترابطها لضمان إدارة فعالة وسلسة لعمليات التوصيل.

## 2. الرؤية والأهداف

**الرؤية:** توفير نظام متكامل لإدارة السائقين يضمن كفاءة عمليات التوصيل، ويحسن تجربة العملاء، ويدعم التوسع الجغرافي للمنصة.

**الأهداف:**

*   إدارة تسجيل السائقين ومراجعة طلبات الانضمام.
*   تتبع حالة السائقين (متاح، مشغول، غير متصل).
*   توزيع المهام على السائقين بشكل فعال.
*   مراقبة أداء السائقين (وقت التوصيل، التقييمات، عدد الطلبات).
*   إدارة معلومات السائقين (المركبة، رخصة القيادة، معلومات الاتصال).
*   توفير تقارير حول أداء السائقين وعمليات التوصيل.

## 3. الأقسام الفرعية والترابط

يتكون قسم السائقين من الأقسام الفرعية التالية، والتي تتفاعل فيما بينها لضمان سير العمليات:

### 3.1. جميع السائقين (All Drivers)

*   **الهدف:** عرض قائمة شاملة بجميع السائقين المسجلين في المنصة، مع إمكانيات البحث، التصفية، وعرض التفاصيل الأساسية لكل سائق.
*   **الترابط:** نقطة انطلاق لإدارة أي سائق. يتكامل مع الأقسام الفرعية الأخرى (Pending Drivers, Active Drivers, Disabled Drivers) لعرض حالة السائق المحددة.

### 3.2. السائقون المعلقون (Pending Drivers)

*   **الهدف:** عرض قائمة بالسائقين الذين قدموا طلبات للانضمام وينتظرون المراجعة والموافقة من قبل المسؤول. يسمح للمسؤول بمراجعة معلومات السائق، وثائق المركبة، واتخاذ قرار بالموافقة أو الرفض.
*   **الترابط:** يتكامل مع قسم التحقق (Business -> Users -> Verification) لضمان استيفاء السائق لمتطلبات التحقق قبل الموافقة.

### 3.3. السائقون النشطون (Active Drivers)

*   **الهدف:** عرض قائمة بالسائقين الذين تم تفعيل حساباتهم ويعملون حالياً مع المنصة. يسمح للمسؤول بمراقبة أدائهم، وتعديل معلوماتهم، أو تعطيلهم إذا لزم الأمر.
*   **الترابط:** يتكامل مع قسم تتبع الشحنات (Operations -> Shipments) لتتبع موقع السائقين النشطين.

### 3.4. السائقون المعطلون (Disabled Drivers)

*   **الهدف:** عرض قائمة بالسائقين الذين تم تعطيل حساباتهم لأسباب مختلفة (مثل عدم الامتثال للسياسات، الأداء الضعيف، أو طلب السائق). يسمح للمسؤول بمراجعة سبب التعطيل، وإعادة تفعيل السائق إذا تم حل المشكلة.
*   **الترابط:** يتكامل مع قسم الدعم (Marketing -> Support) للتعامل مع استفسارات السائقين بخصوص حساباتهم المعطلة.

### 3.5. تقارير أداء السائقين (Driver Performance Reports)

*   **الهدف:** توفير تقارير شاملة حول أداء السائقين، بما في ذلك متوسط وقت التوصيل، عدد الطلبات المكتملة، تقييمات العملاء، والمسافة المقطوعة. يساعد في تقييم فعالية السائقين وتحسين عمليات التوصيل.
*   **الترابط:** يتكامل مع موديول البيانات الذكية (Smart Data -> Reports) لتقديم تقارير مفصلة.

### 3.6. إعدادات السائقين (Driver Settings)

*   **الهدف:** إدارة الإعدادات العامة للسائقين، مثل مناطق التغطية، أوقات العمل، وسياسات الأجور. يسمح للمسؤول بتعديل هذه الإعدادات لضمان كفاءة العمليات.
*   **الترابط:** يتكامل مع قسم الشحن (Operations -> Shipping) لتحديد مناطق التغطية، ومع قسم التحويلات (Financial -> Payouts) لإدارة مدفوعات السائقين.

## 4. الخلاصة

يُعد قسم السائقين عنصراً حيوياً في لوحة تحكم المسؤول، حيث يضمن إدارة فعالة وسلسة لعمليات التوصيل في منصة pazarat. من خلال توفير أدوات شاملة لإدارة السائقين، تتبع أدائهم، وتوزيع المهام، يدعم هذا القسم تجربة العملاء ويساهم في نجاح العمليات اللوجستية للمنصة.
