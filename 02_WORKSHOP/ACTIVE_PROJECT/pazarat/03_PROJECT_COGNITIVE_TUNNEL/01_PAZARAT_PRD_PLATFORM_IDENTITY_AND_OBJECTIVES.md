# Project Type

Pazarat is a large-scale multi-party commerce platform.

It is not a simple online store.

It is not only a marketplace catalog.

It is not only an admin dashboard.

Pazarat is intended to become an operational commerce ecosystem that supports complex commercial activity across multiple user types, business models, workflows, governance layers, and operational systems.

# Relationship Strength

The model should classify relationships when useful.

## Strong Relationship

The section cannot function correctly without the related system.

## Medium Relationship

The section affects or is affected by another system but can be documented separately.

## Weak Relationship

The relationship is indirect or future-oriented.

## Proposed Relationship

The relationship is suggested by platform logic but not yet documented.

This prevents over-linking every system to every other system while still preserving platform coherence.
