# v9.7 Open Neural Operating Kernel

This version restores the original v9 authentic behavior: the runtime does not write a scripted first-contact answer and does not cage the model inside a shortcut. The model is the reasoning engine. The engineer runtime is the silent operating system that provides truth, navigation, governance, and gates.

## Core rule

No first-contact cage. No scripted surface answer.

The sequence is:

1. Understand intent.
2. Enter the control core through `plug-fast` or `open-neural-entry`.
3. Ground the intent in current truth via `file-matrix-pulse` / `truth-sweep-360`.
4. Let the model choose depth according to evidence need, risk, and requested operation.
5. For strategy/write/runtime repair, use `operation-360` and the relevant write/impact gates.
6. Compose the answer naturally from evidence; do not dump packets or describe runtime unless asked.

## Truth rule

File existence is only surface truth. File size is not value. Content value is measured by role, signals, headings, relation to truth spine, query fit, incompleteness, contradiction candidates, and lifecycle impact.

## Anti-patchwork rule

A solution is not accepted because it is locally useful. It must survive:

- parent-goal closure,
- truth sweep,
- missing-ring scan,
- best-fit comparison,
- impact/duplicate gates for writes,
- validation/doctor after execution.

## Runtime boundary

`.engineer` is silent runtime. `02_WORKSHOP/_WORKSHOP_SYSTEM` is method/doctrine. `02_WORKSHOP/ACTIVE_PROJECT` is the active project truth surface.
