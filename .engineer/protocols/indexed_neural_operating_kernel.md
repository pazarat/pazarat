# Indexed Neural Operating Kernel v7

The engineer runtime must not rebuild panoramic context during every request.
It must separate the lifecycle into four phases:

1. **Build Index** — touch current files once and store an operating index with
   fingerprints, semantic value signals, high-value files, and relationship edges.
2. **Fast Plug** — classify intent and risk, load the index, select relevant lenses,
   and return a compact operating packet.
3. **Deep Plug** — only for high-impact work, read owner and linked files selected by
   the index.
4. **Execution Gates** — writing, implementation, or architectural repair requires
   duplicate/impact scans, truth maturity, canon checks, tests, and ledger updates.

The index is a navigation memory, not source truth. Current files remain the source
of truth. A stale index must be rebuilt or treated as unavailable.

This protocol preserves the neural-matrix goal: each operation activates its linked
safeguards without forcing all files into every answer.
