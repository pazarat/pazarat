# Neural Matrix Operating Cycle — v6.14

This protocol turns the central plug from a report generator into an operating nervous system.

## Core idea

The model must not treat scan tools as islands. Every operation activates linked safeguards:

- A claim activates evidence and claim checks.
- A write activates duplicate, impact, write-plan, safe-write, and post-write claim checks.
- An implementation translation activates parent/child reading, state/event alignment, permission/audit checks, data/API gates, and tests.
- External benchmarking activates project-truth anchoring and decision separation.
- Runtime improvement activates doctor, runtime-audit, registry sync, cache policy, and tests.
- Best-fit reasoning activates current-evidence comparison against internal excellence patterns.

## Mandatory entry

Use `plug` or `cycle` before high-impact work. The output is not a reading report. It is the active operating contract.

## Output must guide action

The plug packet must expose:

1. route and scope;
2. tool keyboard;
3. protocol stack;
4. concept matrix;
5. neural reflex graph;
6. evidence gates;
7. mandatory next actions;
8. hard blocks;
9. operation receipt.

## Failure rule

If an operation has a linked reflex and the required gate is missing, the operation is not mature enough to execute.
