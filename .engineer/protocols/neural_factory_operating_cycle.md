# Neural Factory Operating Cycle v9.0

The engineer runtime behaves as a factory, not a loose collection of prompts or tools.

## Cycle

1. **Intent Governance** — classify user intent, risk, and scope.
2. **Central Entry** — enter the operating core, not the project directly.
3. **Truth Surface Selection** — decide whether truth is runtime, workshop, active project, or a combination.
4. **Matrix Touch** — use File Matrix/Runtime Matrix to obtain a fast nervous surface of current files.
5. **Workshop Lens** — apply project standards and maturity method from `_WORKSHOP_SYSTEM`.
6. **Goal Closure** — create a goal contract and capability matrix.
7. **Strategy Simulation** — test the solution lifecycle mentally before execution.
8. **Missing-Ring Scan** — expose missing adapters, validators, contracts, tests, or tools.
9. **Readiness Gate** — return READY_FULL, READY_PARTIAL, or BLOCKED_MISSING_CAPABILITY.
10. **Governed Execution** — execute only through the proper write/tool path.
11. **Validation Doctor** — verify result and runtime health.
12. **Maintenance Ledger** — record decision, gaps, receipts, and follow-up maintenance.

## Key Rule

The cycle may loop. Discovery can reveal that deeper matrix inspection is needed; strategy may send the operation back to truth access; validation may send it back to repair. This back-and-forth is intended and is the nervous-system behavior.
