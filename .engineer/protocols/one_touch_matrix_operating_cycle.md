# One-Touch Matrix Operating Cycle — v6.12

The central operating cycle is the default way an engineer-model enters work.
It is not a report for human reading. It is a steering contract that the model must operate under.

## Cycle

1. `plug <request>`
2. Read route, scope, inspection depth, and tool keyboard.
3. Treat the protocol stack as active constraints.
4. Use the concept matrix to decide what must be read and what must not be guessed.
5. Use evidence gates before any high-impact claim.
6. For writes, run duplicate/impact/write-plan/safe-write dry-run before application.
7. Emit or internally maintain the operation receipt.

## Meaning

- The plug is the central OS entry.
- The matrix is the lens.
- The tool keyboard is the route-specific set of allowed/required controls.
- The receipt is the proof that the operation did not occur by random browsing.

## Non-negotiables

- No assessment from memory.
- No absence claim without zero-result evidence.
- No write without duplicate and impact gates.
- No external source overriding project truth.
- No runtime leakage in project-facing answers.
