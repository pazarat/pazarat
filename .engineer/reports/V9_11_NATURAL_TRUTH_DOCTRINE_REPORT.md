# V9.11 Natural Truth Doctrine Report

## Decision

Use v9.8 as the behavioral baseline because it produced the most natural project-aware response. Do not inherit the v9.9/v9.10 pattern where truth grounding becomes a heavy answer-time cage.

## Correction

Truth is now exposed as an advisory operating compass:

- no `first-contact` template;
- no fixed response shape;
- no mandatory search sequence for ordinary reasoning;
- no artificial time budget;
- file size is not a value criterion;
- the model remains free to choose depth from intent, uncertainty, and risk.

## What changed

- `truth-authority-gate` became an advisory truth compass, not a permission cage.
- `file-matrix-pulse` defaults to all files through safe reader limits, not an arbitrary 220-file cap.
- `plug-fast` now exposes `truth_principles` and claim-strength calibration instead of hard next-action cages.
- `open-neural-entry` now says the runtime calibrates claim strength, not answer shape.

## Intended behavior

The model should feel like an engineer inside a mature operating environment:

1. understand intent;
2. enter the control core;
3. inspect truth surfaces and content value;
4. choose depth freely;
5. answer naturally;
6. calibrate claims to observed evidence;
7. escalate only when the task risk requires it.
