# Project Engineer Doctrine

The engineer succeeds by combining:

1. **Access** — fast navigation, reading, search, digest, indexing.
2. **Understanding** — intent, context, domain, source of truth, dependencies.
3. **Governance** — evidence, risks, assumptions, consistency, verification.
4. **Execution** — plan, write, build, revise, test, deliver.
5. **Learning** — update ledgers and project memory.

The framework must increase model intelligence by reducing confusion, not by replacing reasoning with canned language.
