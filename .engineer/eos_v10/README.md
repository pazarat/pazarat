# EOS v10 Runtime Implementation Scaffold

هذا المجلد هو **التنفيذ الهيكلي الكامل** لمعمارية EOS v10 داخل بيئة المهندس.

## الهدف
توفير بنية تشغيل جاهزة (مجلدات + عقود + مخططات + بايبلاين + وحدات + Doctors) بدل الاكتفاء بوثيقة فلسفية.

## الشجرة
- `kernel/`: هوية التشغيل ونواة التوجيه.
- `contracts/`: عقود الإدخال/الإخراج بين الطبقات.
- `pipeline/`: مراحل التشغيل الرسمية.
- `modules/`: وحدات الحوكمة القابلة للتركيب.
- `doctors/`: فحوصات الصحة الذاتية.
- `indexes/`: فهارس File Matrix والبصمات.
- `schemas/`: مخططات JSON/YAML للتحقق.
- `playbooks/`: Playbooks لتنفيذ الأنماط المتكررة.
- `tests/`: اختبارات ثبات معماري لهذه الطبقة.

## مبدأ التنفيذ
الملفات هنا project-neutral وتعمل كمنظومة تشغيل عامة.
أي معايير خاصة بالمشروع يجب أن تُستخلص وتُحفظ داخل `02_WORKSHOP/ACTIVE_PROJECT`.
