# v6.4 Jet OS Notes

The operating system is not a slogan. It exposes commands that let the model touch the project as a whole without blind sequential reading:

- `matrix`: semantic route + evidence lock.
- `truth-map`: current project truth from content.
- `scenario-map`: scenario/workflow/use-case detection from content.
- `evidence-lock`: claim-ready project truth packet.
- `read-many`: multi-file reading by glob.
- `compare`: compare two evidence directions.
- `duplicate-scan`: detect existing similar material before adding new material.
- `impact-scan`: find related files before editing/writing.
- `write-plan`: generate governance plan before writing.
- `safe-write`: dry-run by default, backup + ledger when applied.
- `claim-check`: reject unsupported or contradicted claims.

The model remains creative, but project claims and writes must be evidence-governed.


The runtime ships without project-specific artifacts. No project name, domain module, vertical slice, or technology stack belongs in the engineer layer. Those facts must be discovered from the active project only.
