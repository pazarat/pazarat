# v6.8 Maturity Truth Notes

This version strengthens the operating system under the answer rather than changing the voice.

Core additions:

- `maturity-map`: compares project areas by semantic evidence, not file size.
- `placeholder-scan`: prevents weak/empty claims without evidence.
- `answer-receipt`: records the route, truth counts, scenario count, top maturity domains, and anti-speculation gates.

Principles:

1. File size is not maturity.
2. Short files may carry high governance or scenario value.
3. A first execution path requires a maturity comparison.
4. The engineer may propose solutions when information is missing, but must label them as proposals, not project facts.
5. The project remains the source of operational truth.
