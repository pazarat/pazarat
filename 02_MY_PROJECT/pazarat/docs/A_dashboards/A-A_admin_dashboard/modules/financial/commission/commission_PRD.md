# وثيقة متطلبات المنتج (PRD): قسم العمولات (Commission Section)

## 1. المقدمة

يُعد قسم العمولات (Commission Section) ضمن موديول المالية (Financial Module) في لوحة تحكم المسؤول هو المسؤول عن إدارة نظام العمولات المعقد في منصة "pazarat". يغطي هذا القسم تحديد نسب العمولات، حسابها، تتبع المدفوعات المستحقة للوكلاء والبائعين، وإدارة سياسات العمولات. تهدف هذه الوثيقة إلى توضيح الأهداف العامة لهذا القسم، أقسامه الفرعية، وكيفية ترابطها لضمان إدارة مالية شفافة وعادلة للعمولات.

## 2. الرؤية والأهداف

**الرؤية:** توفير نظام آلي ودقيق لإدارة العمولات يضمن الشفافية والعدالة لجميع الأطراف (المنصة، البائعين، الوكلاء)، ويحفز الأداء، ويسهل عملية التدقيق المالي.

**الأهداف:**

*   تحديد وإدارة أنواع مختلفة من العمولات (نسبة مئوية، مبلغ ثابت، عمولة على المنتج/الخدمة).
*   حساب العمولات المستحقة للوكلاء والبائعين بشكل آلي ودقيق.
*   تتبع حالة دفع العمولات (مستحقة، مدفوعة، معلقة).
*   توفير تقارير شاملة حول العمولات المدفوعة والمستحقة.
*   إدارة سياسات العمولات وتطبيقها على المنتجات والبائعين والوكلاء.
*   ضمان الامتثال للمتطلبات المحاسبية والضريبية المتعلقة بالعمولات.

## 3. الأقسام الفرعية والترابط

يتكون قسم العمولات من الأقسام الفرعية التالية، والتي تتفاعل فيما بينها لضمان سير العمليات:

### 3.1. جميع العمولات (All Commissions)

*   **الهدف:** عرض قائمة شاملة بجميع العمولات المحتسبة في المنصة، مع إمكانيات البحث، التصفية، وعرض التفاصيل الأساسية لكل عمولة (المالك، المبلغ، المصدر، الحالة).
*   **الترابط:** نقطة انطلاق لتتبع أي عمولة. يتكامل مع الأقسام الفرعية الأخرى (Pending Commissions, Paid Commissions) لعرض حالة العمولة المحددة.

### 3.2. العمولات المستحقة (Pending Commissions)

*   **الهدف:** عرض قائمة بالعمولات التي تم احتسابها ولكن لم يتم دفعها بعد. يسمح للمسؤول بمراجعة هذه العمولات واتخاذ قرار بالدفع.
*   **الترابط:** يتكامل مع قسم التحويلات (Financial -> Payouts) لبدء عملية دفع العمولات.

### 3.3. العمولات المدفوعة (Paid Commissions)

*   **الهدف:** عرض قائمة بالعمولات التي تم دفعها بنجاح للوكلاء أو البائعين. يسمح للمسؤول بتأكيد الدفع وتتبع سجل المدفوعات.
*   **الترابط:** يتكامل مع قسم المعاملات (Financial -> Transactions) لتسجيل حركة دفع العمولات.

### 3.4. إعدادات العمولات (Commission Settings)

*   **الهدف:** إدارة وتكوين قواعد احتساب العمولات. يشمل ذلك تحديد نسب العمولات الافتراضية، القواعد الخاصة بمنتجات أو بائعين محددين، وشروط الدفع.
*   **الترابط:** يتكامل مع قسم المنتجات (Business -> Products) لتطبيق قواعد العمولات على المنتجات، ومع قسم التسويق بالعمولة (Business -> Affiliate) لتحديد عمولات الوكلاء.

### 3.5. تقارير العمولات (Commission Reports)

*   **الهدف:** توفير تقارير تحليلية حول أداء العمولات، مثل إجمالي العمولات المدفوعة، العمولات المستحقة، الوكلاء/البائعين الأكثر تحقيقاً للعمولات. يساعد في تقييم فعالية برنامج العمولات.
*   **الترابط:** يتكامل مع موديول البيانات الذكية (Smart Data -> Reports) لتقديم تقارير مفصلة.

## 4. الخلاصة

يُعد قسم العمولات عنصراً حاسماً في لوحة تحكم المسؤول، حيث يضمن إدارة مالية شفافة وعادلة لجميع العمولات في منصة pazarat. من خلال توفير أدوات شاملة لتحديد، حساب، وتتبع العمولات، يدعم هذا القسم تحفيز الأداء ويساهم في النمو المستمر للمنصة.
