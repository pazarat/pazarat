# وثيقة متطلبات المنتج (PRD): قسم الوكلاء (Agents Sub-Section)

## 1. المقدمة

يُعد قسم "الوكلاء" (Agents Sub-Section) ضمن قسم المستخدمين (Users Section) في موديول الأعمال (Business Module) بلوحة تحكم المسؤول، هو النقطة المخصصة لإدارة الوكلاء أو المسوقين بالعمولة الذين يروجون لمنتجات المنصة أو منتجات البائعين. تهدف هذه الوثيقة إلى توضيح الأهداف العامة لهذا القسم الفرعي، وظائفه، وكيفية تفاعله مع الأقسام الأخرى لضمان إدارة شاملة وفعالة لبرامج التسويق بالعمولة.

## 2. الرؤية والأهداف

**الرؤية:** توفير بيئة متكاملة لإدارة الوكلاء، بدءًا من عملية التسجيل والتحقق، مرورًا بتتبع أدائهم وحساب عمولاتهم، وصولًا إلى توفير الأدوات اللازمة لنجاحهم، مما يدعم نمو برنامج التسويق بالعمولة ويزيد من انتشار المنتجات.

**الأهداف:**

*   عرض وإدارة قائمة الوكلاء المسجلين في المنصة.
*   مراجعة طلبات الانضمام كوكيل وتفعيل/تعطيل حساباتهم.
*   توفير أدوات للبحث والتصفية المتقدمة للوكلاء.
*   عرض تفاصيل شاملة لكل وكيل (معلومات شخصية، معلومات الاتصال، المنتجات المروجة، أداء الحملات، العمولات المستحقة والمدفوعة).
*   إتاحة إجراءات سريعة لإدارة حسابات الوكلاء (مثل حظر، إلغاء حظر، إرسال رسالة).
*   تتبع أداء الوكلاء ومساهمتهم في المنصة.

## 3. الوظائف الرئيسية والترابط

### 3.1. عرض قائمة الوكلاء (Agents List Display)

*   **الهدف:** عرض جدول تفاعلي يحتوي على قائمة بجميع الوكلاء. يجب أن يتضمن الجدول أعمدة قابلة للتخصيص مثل: معرف الوكيل، الاسم الكامل، البريد الإلكتروني، رقم الهاتف، حالة الحساب (نشط، معلق، محظور)، تاريخ التسجيل، عدد المنتجات المروجة، إجمالي العمولات.
*   **الترابط:** يتكامل مع قاعدة بيانات المستخدمين لجلب البيانات، ومع قسم التسويق بالعمولة (Business -> Affiliate) لعرض معلومات الحملات.

### 3.2. البحث والتصفية (Search & Filtering)

*   **الهدف:** تمكين المسؤولين من العثور على وكلاء محددين بسرعة باستخدام حقول بحث متعددة (مثل البحث بالاسم، البريد الإلكتروني، رقم الهاتف) وخيارات تصفية متقدمة (مثل التصفية حسب حالة الحساب، عدد المنتجات المروجة، نطاق العمولات، تاريخ التسجيل).
*   **الترابط:** يتكامل مع وظيفة عرض قائمة الوكلاء لتحديث النتائج ديناميكياً.

### 3.3. عرض تفاصيل الوكيل (Agent Details View)

*   **الهدف:** عند النقر على أي وكيل في القائمة، يتم عرض صفحة تفاصيل شاملة للوكيل. هذه الصفحة يجب أن تحتوي على معلومات شخصية، معلومات الاتصال، قائمة المنتجات المروجة، سجل الحملات، تقارير الأداء، سجل المعاملات المالية للعمولات، وحالة التحقق.
*   **الترابط:** يتكامل مع قسم التحقق (Users -> Verification) لعرض حالة التحقق، ومع قسم التسويق بالعمولة (Business -> Affiliate) لعرض حملات الوكيل، ومع قسم العمولات (Financial -> Commission) لعرض معاملاته المالية.

### 3.4. إجراءات الوكيل (Agent Actions)

*   **الهدف:** توفير أزرار أو قوائم سياقية لإجراءات سريعة على الوكلاء، مثل: تفعيل/تعطيل الحساب، حظر/إلغاء حظر الوكيل، إرسال رسالة، تعديل معلومات الملف الشخصي، أو حذف الحساب (مع تأكيد).
*   **الترابط:** تتكامل مع قسم الإشعارات (System -> Notifications) لإرسال الرسائل.

### 3.5. مراجعة طلبات الانضمام (Onboarding Requests Review)

*   **الهدف:** توفير واجهة لمراجعة طلبات الوكلاء الجدد للانضمام إلى المنصة. يشمل ذلك عرض معلومات الوكيل، الوثائق المقدمة، واتخاذ قرار بالموافقة أو الرفض.
*   **الترابط:** يتكامل مع قسم التحقق (Users -> Verification) لمراجعة الوثائق، ومع قسم الموافقات (Users -> Approved) لتسجيل قرار الموافقة.

## 4. المرجع البصري (UI/UX Reference)

(سيتم توليد صورة مرجعية هنا لاحقاً توضح تصميم صفحة "الوكلاء"، بما في ذلك تخطيط الجدول، حقول البحث والتصفية، وأزرار الإجراءات، وصفحة تفاصيل الوكيل.)

## 5. الخلاصة

يُعد قسم "الوكلاء" عنصراً حيوياً في إدارة المستخدمين، حيث يوفر للمسؤولين رؤية شاملة وأدوات قوية للتحكم في شريحة الوكلاء والمسوقين بالعمولة. من خلال توفير إمكانيات بحث وتصفية متقدمة، وعرض تفاصيل شاملة، وإجراءات سريعة، يضمن هذا القسم إدارة فعالة ومرنة للوكلاء ويساهم في نمو برنامج التسويق بالعمولة ونجاح المنصة.
